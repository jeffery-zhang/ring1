# AGENTS.md 指导文件

## 目标

- 在这里我们的目标是开发 cli 工具

## 你的行为准则

- **保证可追溯的 git 记录** 每一步任务执行完成都要 commit 代码, 简单概述一下完成的工作, 用中文
- **保持极简设计** 永远不要做多余的事
- **谨慎行动** 当一件事不是100%确定时, 问你的人类

## 工作流

### 设计文档

- 当 Agent 根据需求给出设计方案时, 需要记录到 memory/designs 目录下的 md 文件中
- 一个版本一个设计方案, 文件名称: <项目名称>-<版本号>.md

#### 版本号说明

- 设计文档始终用 vx.x.x 标记版本, 对应 major, minor, patch
- 所有设计文档都要保留, 更新版本时新版本文档仅描述新功能和对旧功能的改造

### 实施计划/步骤

- 实施计划需严格按照对应设计文档给出, 并需要记录到 memory/plans 目录下的 <MM-DD-HH-mm-ss>-<对应设计文档名称>-实施计划.md 文件中
- 计划需要模块化区分, 每个模块保持低耦合, 模块内再细分执行步骤
- 每个模块和步骤都要有完成状态标记, 开发时每完成一个步骤都要打勾

### 问题修复记录

- 当验证阶段发现问题, 需要将问题记录到 memory/validations 目录下的 <MM-DD-HH:mm:ss>-<问题摘要>.md 文件中
- 问题记录需要包含完整上下文, 并列出详细的修复步骤(带状态)
- 执行修复步骤后要给对应步骤打勾
